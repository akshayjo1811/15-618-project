Summary

We will develop a 2D heat diffusion simulation using finite difference methods, implement parallel versions using both shared address space and message passing programming models, and conduct a detailed performance analysis comparing their scalability, efficiency, and execution characteristics on multi-core CPU clusters available in the GHC lab.

Background

Heat diffusion is a fundamental physical phenomenon described by the heat equation:

∂T/∂t = α(∂²T/∂x² + ∂²T/∂y²)

The finite difference method (FDM) is used to solve this equation numerically by discretizing the domain into a grid.

The Challenge

Each grid point update depends on its neighbors, requiring communication at subdomain boundaries. This introduces synchronization overhead. We will explore both shared and distributed memory approaches and analyze tradeoffs in performance and accuracy.

Resources

We will build the simulator from scratch but may adapt existing C/C++ models if useful. Research papers on heat diffusion will be referenced.

Goals and Deliverables

  • Implement a sequential 2D heat diffusion simulation.
  • Develop shared memory parallelization (OpenMP) and distributed memory parallelization (MPI).
  • Analyze communication overhead, scaling efficiency, and synchronization costs.
  • Explore a hybrid OpenMP+MPI approach.

Platform Choice

X86-64 multi-core CPU architecture with OpenMP & MPI libraries. Implementation in C/C++.

Rough Schedule

  • Week 1: Setup, finalize approach, sequential implementation.
  • Week 2: OpenMP implementation.
  • Week 3: MPI implementation.
  • Week 4: Hybrid MPI + OpenMP implementation and tradeoff analysis.
  • Week 5: Benchmark testing and final report.
